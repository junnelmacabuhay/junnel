## 💡 Pro Tips

### Tip 1: Use Specs First
Write `.spec.js` files BEFORE tests. Document requirements first!

### Tip 2: Follow AAA Pattern
Every test: Arrange → Act → Assert

### Tip 3: Keep Tests Isolated
Each test should work independently. No shared state!

### Tip 4: Mock External Dependencies
Only mock outside services. Never mock what you're testing!

### Tip 5: Descriptive Names
Test names should explain what's being tested, not how.
